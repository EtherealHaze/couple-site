## 踩坑复盘

### 1. JSON 并发写入覆盖
**问题**：前端两个操作几乎同时触发 `/api/save`，后一次写入覆盖前一次修改。出现场景：快速连续勾选两个心愿清单项。
**解决**：前端增加 300ms 防抖，合并连续保存请求；服务端采用"接收完整 key-value → 合并到内存对象 → 全量写回"策略。

### 2. Base64 大图导致 data.json 膨胀
**问题**：用户上传 4K 高清照片，Base64 编码后单张可达 10MB+，data.json 迅速膨胀到 50MB+，加载/保存均超时。
**解决**：改为 Multer 文件上传，图片存到 `uploads/` 目录，data.json 仅存相对路径；前端增加 Canvas 压缩逻辑。

### 3. 定时器未清理内存泄漏
**问题**：计时器 setInterval 在页面切换后持续运行，开发工具 Memory 面板显示定时器数量持续增长。
**解决**：使用 `visibilitychange` API 在页面不可见时暂停计时，统一管理所有定时器 ID 便于清理。

### 4. 手机浏览器中文乱码
**问题**：iOS Safari 访问局域网地址时部分中文显示为乱码。
**解决**：检查并统一所有文件的编码为 UTF-8；Express 响应头显式声明 `charset=utf-8`；HTML meta 标签声明 charset。

### 5. Multer 文件名冲突
**问题**：Multer 默认生成随机文件名但无扩展名，浏览器无法识别图片类型。
**解决**：上传后使用 `fs.renameSync` 添加原始文件扩展名，返回带扩展名的 URL。

### 6. CORS 与局域网访问
**问题**：手机访问时部分资源加载失败，浏览器报 CORS 错误。
**解决**：Express 监听 `0.0.0.0` 而非 `localhost`，使用相对路径引用所有静态资源，避免跨域问题。
