# 💕 Couple Site — 情侣专属互动Web平台

> 属于两个人的私密在线空间：时光轴、照片墙、悄悄话、心愿清单、我们的收藏

## 📖 项目简介

Couple Site 是一个面向情侣的私密互动 Web 平台，采用 Node.js + 原生 JavaScript SPA 架构。通过暗号验证进入专属空间后，双方可以共同编辑时光轴故事、上传照片到共享画廊、互写悄悄话、勾选心愿清单、管理共同收藏（歌曲/电影/美食/旅行地）。

项目设计为局域网部署使用（手机/电脑均可访问），无需注册账号，通过约定的暗号验证身份，所有数据以 JSON 文件持久化存储，简洁轻量。

**开发背景**：大二软件工程个人实战项目，学习 Node.js 后端开发、原生 JavaScript SPA 架构、前后端数据交互、文件上传处理。

## 🛠️ 技术栈

| 类别 | 技术 |
|------|------|
| 后端 | Node.js + Express 4.x |
| 前端 | 原生 JavaScript (SPA)、HTML5、CSS3 |
| 数据存储 | JSON 文件持久化（data/data.json） |
| 文件上传 | Multer（multipart/form-data） |
| 运行时 | Node.js 18+ |

## ✨ 核心功能

1. **🔐 暗号验证登录** — 两步验证：选择身份（他/她）→ 输入两人约定的暗号
2. **⏱️ 在一起计时器** — 实时显示在一起的天/时/分/秒，纪念日配置
3. **📖 时光轴故事** — 共同编辑恋爱里程碑事件，支持详细记录（照片、评分、备注）
4. **🖼️ 共享照片画廊** — 6 宫格照片墙，支持本地上传和点击查看大图
5. **💌 悄悄话留言板** — 他/她分别留言，卡片式展示
6. **📝 心愿清单** — 共同维护待完成事项，勾选完成触发撒花动画
7. **⭐ 我们的收藏** — 共同编辑歌曲/电影/美食/旅行地四大收藏夹
8. **⚙️ 设置面板** — 修改昵称、暗号、纪念日、编辑时光轴和收藏
9. **🎉 撒花特效** — 重要操作（登录成功/完成任务）触发全屏 emoji 庆祝动画

## 🚀 本地启动步骤

### 环境准备
```bash
# 确保已安装 Node.js 18+
node -v
```

### 安装与启动
```bash
# 1. 进入项目目录
cd couple-site

# 2. 安装依赖
npm install

# 3. 启动服务器
npm start
```

### 访问
- **本机访问**：http://localhost:3000
- **局域网访问**：http://[你的局域网IP]:3000（手机浏览器输入此地址）
- 查看局域网 IP：命令行运行 `ipconfig` 查看 IPv4 地址

### 首次使用
1. 打开页面后，点击"设置"（导航栏齿轮图标）
2. 设置两人的名字、暗号、纪念日
3. 输入暗号验证进入主界面
4. 开始在时光轴、画廊、留言板互动

## 🔧 开发难点与解决方案

### 1. JSON 并发存储丢失问题
**难点**：多个请求同时修改 data.json 时，后写入的请求可能覆盖前面的修改，导致数据丢失。
**方案**：采用"读-改-写"全量保存策略。由于项目规模小、并发量低，当前方案是每次保存时将前端传来的完整 key-value 合并到内存对象后全量写回文件。生产环境建议引入简单的写入队列或使用 SQLite。

### 2. Base64 大图性能问题
**难点**：用户上传的高清照片以 Base64 编码存储在 JSON 中，单张图片可达数 MB，导致 data.json 文件膨胀、页面加载卡顿。
**方案**：
- 前端上传时使用 FileReader 读取并在 `<canvas>` 上压缩（限制最大宽度 800px，JPEG 质量 0.6）
- 服务端使用 Multer 将文件存到 `data/uploads/`，JSON 中仅存储文件路径而非 Base64 数据
- 后续版本可进一步引入图片懒加载和缩略图机制

### 3. 定时器内存泄漏
**难点**：在一起计时器使用 `setInterval` 每秒更新，页面切换或组件销毁时未清理定时器，导致内存泄漏和不必要的 CPU 占用。
**方案**：
- 统一管理所有定时器 ID，在页面不可见时使用 `visibilitychange` 事件暂停计时
- 组件销毁前调用 `clearInterval` 清理
- 计时器回调中避免创建大量 DOM 对象

### 4. 局域网中文乱码问题
**难点**：手机浏览器通过局域网访问时，部分中文内容显示乱码。
**方案**：
- HTML 头部显式声明 `<meta charset="UTF-8" />`
- Express 响应设置 `Content-Type: application/json; charset=utf-8`
- 所有文件保存/读取使用 UTF-8 编码
- data.json 使用 `JSON.stringify` 的 `ensure_ascii: false` 保持中文可读性

### 5. 暗号验证的前端安全性
**难点**：暗号存储在 data.json 中，通过 API 加载到前端验证，存在一定的安全风险。
**方案**：
- 暗号在后端做验证而非前端（虽然当前实现中 settings 包含 secret，但验证逻辑在前端）
- 后续可改为后端 API 验证：`POST /api/verify` 接收身份+暗号，服务端比对后返回 token
- 生产环境部署建议配置 HTTPS 和简单的 rate limiting

### 6. SPA 单页架构的状态管理
**难点**：所有功能模块在单个 HTML 页面中，全局变量管理混乱，模块间数据同步困难。
**方案**：
- 使用全局 `appData` 对象作为唯一数据源
- 通过 `apiSavePartial` 统一保存接口，每次修改后调用对应模块的 `render*` 函数刷新 UI
- 将不同功能模块的代码在 JS 文件中用注释分隔，提高可维护性

- 原生 JavaScript DOM 操作与事件处理
- CSS3 动画（撒花特效、渐变、过渡）
- 响应式布局与移动端适配
- 局域网部署与跨设备访问
